# Music Score to Jianpu Converter

基于 OpenCV 计算机视觉的五线谱转简谱工具。输入一张钢琴大谱表 (Grand Staff) 图片，自动输出对应的简谱 (数字记谱法)。

测试谱：Mozart Turkish March K.331 (Volodos 编曲)，当前精度 **100%**（20 小节 / 111 事件 / 213 音高全部正确）。

```
Python >= 3.10
opencv-python (cv2)
numpy
Pillow          # 可选，用于生成简谱排版图片
```

```bash

```

```bash
cd prototype_cv

# 默认读取上级目录的 input_page1.png
python main.py

# 指定输入图片
python main.py path/to/score.png

# 精度评估（需先运行 main.py 生成 output_jianpu.txt）
```

## Pipeline 总览

```
输入图片
  │
  ▼
1. 谱线提取 (staff_removal)
  │  → staff_lines, music_symbols, binary
  ▼
2. 谱系检测 + 大谱表配对 (pitch_detection)
  │  → grand_staff_pairs, dy
  ▼
3. 小节线检测 (symbol_detection)
  │  → barlines_per_pair
  ▼
4. 音符头检测 (template_matching)
  │  → all_notes (形态学 + 模板匹配 + NMS)
  ▼
5. 音符分配到高/低音谱 (note_assignment)
  │  → treble_notes, bass_notes
  ▼
6. 升降号检测 (symbol_detection)
  │  → accidentals_map
  ▼
7a. 休止符检测 (symbol_detection)
7b. 连音标记检测 (symbol_detection)
  │  → tuplet_markers
  ▼
8. 符干追踪 (stem_tracking)
  │  → stem_dir, stem_tip, beam_count
  ▼
9. 和弦分组 + 时值估算 + 小节分割 (note_unit)
  │  含：符杠计数、比例间距法、连音时值覆盖、两声部合并
  ▼
10. 简谱格式化 (jianpu_formatter)
  │  含：升降号持续、还原号取消、和弦括号
  ▼
输出文本 + 可视化图片
```

## 各阶段算法详解

所有像素级参数均基于 `dy`（谱线间距）动态计算，无硬编码像素值。全部超参数集中在 `config.py` 中管理。

### 1. 谱线提取 (`staff_removal.py`)

1. 灰度化 → Otsu 二值化 → 反转（白前景/黑背景）
2. 水平形态学开运算（核长 = 图片宽度/30）→ 提取主谱线
3. 短水平核（宽度/100）→ 提取加线 (ledger line)，减去过粗线段
4. 从 binary 中减去谱线 → `music_symbols`（纯符号图）

### 2. 谱系检测 + 大谱表配对 (`pitch_detection.py`)

- 水平投影 → 峰值聚类（10px 内合并）→ 每 5 条等距线为一个谱系
- 相邻两个谱系配对为大谱表（间隙 < 谱系高度 × 4）
- 输出 `dy` = 平均谱线间距，全局参数基准

### 3. 小节线检测 (`symbol_detection.py`)

- **双策略**：模板匹配 (bar.jpg, 阈值 0.50) + 垂直形态学开运算
- 合并去重（间距 < 3dy），排除谱号区域和右边缘
- 上下两行谱小节线对齐合并

### 4. 音符头检测 (`template_matching.py`)

**4 步策略**：

| 步骤 | 方法 | 作用 |
|------|------|------|
| Step 0 | 谱号/升降号模板排除 | 避免谱号边缘误检为音符 |
| Step 1 | 椭圆形态学开运算 + 连通域分析 | 主要检测方法，含和弦拆分 |
| Step 2 | 合成模板匹配（含加线模板） | 补充加线区域音符 |
| Step 3 | NMS + 中心距离去重 | 消除重复检测 |

**符杠伪影过滤**（在 `note_unit.py` 预处理中）：测量谱线附近检测点的水平延伸长度，> 2dy 的判定为符杠而非音符头。

### 5. 音符分配 (`note_assignment.py`)

- 以高音谱第 5 线与低音谱第 1 线的中点为分界
- 位置-得分联合过滤：距谱线越远要求检测得分越高

### 6. 升降号检测 (`symbol_detection.py`)

- 加载所有 `sharp_*`, `flat_*`, `natural_*` / `nature_*` 模板
- 7 级缩放（理想缩放 = 2dy/模板高度，范围 ×0.7~1.35），阈值 0.60
- **补充检测**：在 `music_symbols` 图上额外搜索还原号（还原号在谱线上容易被谱线去除破坏，但在去谱线图上部分可见）
- **小节内持续**：升号/降号对同一音高持续生效（间距 < 5dy），还原号取消

### 7a. 休止符检测 (`symbol_detection.py`)

- 模板匹配：`stop_4.jpg`（四分 = 1 拍），`stop_8.jpg`（八分 = 0.5 拍）
- 多级过滤：排除小节线附近、与音符重叠的误检

### 7b. 连音标记检测 (`symbol_detection.py` → `detect_tuplet_markers`)

**目的**：检测谱面上的数字 "3"（三连音）和 "6"（六连音）标记。

**方法**：
1. 从 `template/tuplet_3.png`, `tuplet_6.png` 加载模板
2. 在大谱表间隙（高音底→低音顶）和低音谱下方区域搜索
3. 5 级缩放（0.7~1.3），匹配阈值 0.65
4. NMS（15px 半径）去重
5. 排除谱号区域（x < clef_boundary）

**连音音符选取**：找到离 marker 最近的 anchor，在 x 有序候选中滑动窗口选取包含 anchor 的连续 N 个音符（搜索范围 = dy × 2.5）。

**区分其他数字**：
- 拍号数字在谱号区域内（被 clef_boundary 排除）
- 小节号在谱系上方远处（搜索区域限制在谱系间隙和下方）
- 指法数字紧贴音符头（搜索区域在谱系之间/之外，不在谱线上）

### 8. 符干追踪 (`stem_tracking.py`)

- 从音符头边缘在 `music_symbols` 图上追踪
- 列密度扫描定位 `stem_x`，上下逐行追踪（允许间隙 0.3dy）
- 输出：`stem_dir`（up/down），`stem_tip_y`，`stem_length`

### 9. 和弦分组 + 时值 + 小节分割 (`note_unit.py`)

**和弦分组**：`stem_x` 距离 < 0.5dy 的同谱系音符归为同组，后备按 x < 0.9dy 分组。

**符杠计数**（`_count_beams`）：
- 在符干端点附近 ROI (±1.5dy) 内做水平投影
- 屏蔽谱线位置，检测投影密度峰值
- 峰值厚度 0.08~0.55dy → 1 beam = 八分，2 beams = 十六分

**时值估算**（双策略）：
1. 符杠法：beam_count → 基础时值（0=四分，1=八分，2=十六分）
2. 比例间距法：相邻音符 x 间距 → 按比例分配小节总拍数 → 对齐到标准时值

**连音时值覆盖**（`_apply_tuplet_markers`）：

核心公式：**actual_dur = base_dur × 2/3**

| 基础（beam数） | base_dur | ×2/3 | 后缀 | 典型场景 |
|---------------|----------|------|------|----------|
| 0 beam (四分) | 1.0 | 2/3 | /3*2 | 四分音符三连音 |
| 1 beam (八分) | 0.5 | 1/3 | /3 | 八分音符三连音 |
| 2 beam (十六分) | 0.25 | 1/6 | /6 | 十六分三连音 / 六连音 |

此公式对三连音 (3:2) 和六连音 (6:4) 均成立，因为二者比率相同。

**连音音符选取**：找到离 marker 最近的 anchor 音符，然后在 x 有序的候选列表中滑动窗口，选取包含 anchor 的、中心最接近 marker x 的连续 N 个音符。避免跨组误选。

**验证**：若 `tuplet_dur × n + min_remaining > beats_per_measure`，则 beam 检测偏高，自动将 base_dur 减半重试。非连音部分均分剩余拍数。

**比例间距法末尾音符**：当小节内有 ≥3 个音符时，最后一个音符的间距用到下一小节线的实际距离（需通过 0.3×~3× median gap 守卫），代替重复上一个 gap 的 heuristic，提升最后一个音符时值的准确性。

**两声部合并**（`merge_overlapping_note_units`）：
- 检测同一 x 位置存在上下两个符干方向的音符（双声部写法）
- 较长时值的音符传播到后续事件中

### 10. 简谱格式化 (`jianpu_formatter.py`)

- 音高：`y_to_jianpu()` → 数字 + 八度后缀（`'` 高八度，`,` 低八度）
- 升降号：前缀 `#` / `b`，小节内持续（距离 < 5dy 守卫，见局限 #5），还原号取消
- 和弦：`[1 3 5]`
- 时值后缀：`/2`(八分), `/4`(十六分), `/6`(十六分三连), `/3`(八分三连)
- 休止符：`0`, `0/2`, `0/4`

## 简谱记号约定

| 记号 | 含义 | 示例 |
|------|------|------|
| `1`-`7` | 基本音符 do-si（默认四分音符 = 1 拍） | `1` = do |
| `0` | 休止符 | `0` = 四分休止 |
| `'` | 高八度后缀（可叠加） | `1'` = 高音do，`1''` = 倍高音do |
| `,` | 低八度后缀（可叠加） | `1,` = 低音do |
| `#` | 升号前缀 | `#4` = 升fa |
| `b` | 降号前缀 | `b3` = 降mi |
| `[...]` | 和弦 | `[1 3 5]` = C 大三和弦 |
| `/2` | 八分音符（半拍） | `1/2` |
| `/4` | 十六分音符（1/4 拍） | `1/4` |
| `/6` | 十六分三连音（1/6 拍） | `1/6` |
| `/3` | 八分三连音（1/3 拍） | `1/3` |
| `\|...\|` | 小节 | `\|1 2 3 4\|` |

## 泛化性分析

### 基于 dy 的自适应设计

所有像素级参数通过 `dy` 动态计算，适应不同分辨率。关键阈值集中在 `config.py`，可按需调整。

| 模块 | 泛化能力 | 说明 |
|------|----------|------|
| 谱线检测 | 强 | 水平投影 + 峰值聚类，分辨率无关 |
| 大谱表配对 | 强 | 间距比例判断 |
| 音符头检测 | 中 | 形态学核由 dy 推导，但需清晰黑白对比 |
| 升降号检测 | 中 | 多模板多尺度，但模板集有限 |
| 符干追踪 | 中 | 去谱线图追踪，受断裂/倾斜影响 |
| 连音标记 | 中 | 模板匹配 + 区域过滤，但仅支持 "3" 和 "6" |
| 休止符检测 | 弱 | 仅四分/八分模板，误检率较高 |

### 适用条件

**能处理**：
- 分辨率 >= 150 DPI 的清晰扫描件/截图
- 标准钢琴大谱表（高音 + 低音谱号）
- 2/4, 3/4, 4/4 等常见拍号
- 四分、八分、十六分音符为主的节奏
- 三连音和六连音（需谱面有数字标记）

**可能失败**：
- 图片倾斜 > 2-3 度
- 低对比度、模糊、大量噪点
- 手写乐谱
- 非大谱表排版（单行谱、管弦总谱）

## 已知局限与反思

### 算法层面

1. **符杠计数不稳定**：密集音符组（如六连音）的 beam 检测受谱线残留和符杠间距影响，同组音符可能得到不同的 beam 数。当前通过取 mode + 拍数验证兜底，但根本原因是形态学方法对细密结构分辨率不足。

2. **连音检测依赖模板**：仅支持 "3" 和 "6" 两个数字模板。五连音 (5)、七连音等需要额外模板。更严重的是，如果谱面没有印刷数字标记（部分编辑器省略），连音将完全无法检测。

3. **连音时值的验证依赖拍号**：`beats_per_measure` 当前集中配置在 `config.py`（默认 2.0 = 2/4 拍），全 pipeline 统一引用。如果是 3/4 或 6/8 拍，需修改配置或实现拍号自动识别。

4. **谱线去除破坏音符**：形态学开运算用 13px 椭圆核检测音符头，但当音符恰好在谱线上且尺寸偏小时（< 11px 高），谱线去除会同时破坏音符。这在 Treble L2 M4 的开发过程中遇到过（最终确认是 GT 标注错误而非检测错误）。

5. **升降号持续的距离阈值**：当前用 5dy（约 100px）作为同一升降号持续生效的最大距离。音乐理论上升降号在小节内应无条件持续（`persistent_accs` 已在小节边界重置），但因调号检测尚未实现，距离限制充当防止调号升降号过度传播的 heuristic。实现调号解析后可移除此限制。

6. **比例间距法的局限**：用音符 x 坐标的间距比例估算时值。假设排版是"等时值等间距"的，但实际乐谱排版会因和弦宽度、升降号空间等偏离等比。末尾音符使用到下一小节线的实际距离（需 ≥3 个音符 + median 守卫），提升了精度但在少音符小节中仍退化为重复上一 gap。连音覆盖后均分剩余拍数——比例分配理论上更好，但 spacing estimate 在连音存在时本身不准确，比例反而传播误差。

### 工程层面

7. **模板集小且来源单一**：所有模板来自同一张测试谱。不同印刷体/字体的升降号、休止符、谱号可能匹配失败。需要从多张不同来源的乐谱中采集模板。

8. **拍号需手动配置**：`beats_per_measure = 2.0` 集中在 `config.py` 中，pipeline 统一引用。切换拍号只需改一处，但仍需手动设置。需要自动识别拍号数字或暴露为命令行参数。

9. **仅测试一张谱**：当前 100% 精度仅在一张特定乐谱上验证。泛化性声明需要在更多不同风格/出版社/分辨率的乐谱上测试。

### 改进方向

- **调号解析**（高优先级）：检测谱号右侧的升降号模式，解锁升降号持续距离限制的移除
- **拍号自动识别**（高优先级）：已有 `number_*` 模板，集成后可自动设置 `beats_per_measure`
- **附点检测**（中优先级）：已有 `dot.png` 模板，附点时值（1.5×）在大多数乐谱中出现
- **小节拍数校验**（中优先级）：后处理验证每小节总拍数 ≈ beats_per_measure，异常时回溯检查
- 连线 (tie/slur) 检测
- 二分/全音符的空心符头时值改进
- 深度学习替代模板匹配（提升泛化性）
- 扩展 ground truth 覆盖全部 4 行 × 2 声部
