# Music Score to Jianpu (简谱) Converter

基于 OpenCV 计算机视觉的五线谱转简谱工具。输入一张钢琴大谱表(Grand Staff)图片，自动输出对应的简谱(数字记谱法)。

## 功能概述

- 高音谱号 + 低音谱号双行大谱表识别
- 和弦识别 `[1 3 5]`
- 升降号检测 `#5`, `b3`，含小节内持续效果与还原号取消
- 休止符识别（四分、八分休止符）
- 时值推断（四分、八分、十六分音符），基于符杠检测 + 间距比例双重策略
- 加线(ledger line)音符检测，超出五线谱范围的高低音
- 弱起小节(anacrusis)自动识别
- 自动评估：与人工标注 ground truth 对比，输出逐小节精度报告

## 算法详解

整个流程分为 10 个阶段，按顺序执行。所有参数均基于 `dy`（谱线间距，像素）动态计算，无硬编码像素值。

### 第 1 阶段：谱线提取 (`staff_removal.py`)

**输入**：原始图片路径

**方法**：
1. 灰度化 → 大津法二值化 → 反转（黑底白字）
2. **主谱线提取**：水平形态学开运算，核长度 = `图片宽度 / 30`（只保留足够长的水平线段）
3. **加线(ledger line)提取**：较短水平核 = `图片宽度 / 100`，再减去过粗水平线（膨胀后差集），仅保留细短线段
4. 合并主谱线与加线 → `staff_lines` 掩码
5. 从二值图中减去谱线 → `music_symbols`（纯符号图）

**输出**：`(staff_lines, music_symbols, binary)` 三张二值图


### 第 2 阶段：谱系检测 + 大谱表配对 (`pitch_detection.py`)

**谱系检测** `get_staff_systems()`：
1. 对 `staff_lines` 进行水平投影（每行白色像素求和）
2. 峰值阈值 = `最大投影值 × 0.3`，提取所有超过阈值的行号
3. 将相邻 10px 内的行号聚类为一条线 → 得到所有谱线的 y 坐标列表
4. 每 5 条等距线为一组（间距均匀性检查：`最大间距 - 最小间距 < 平均间距 × 0.8`）→ 若不满足则跳过该候选

**大谱表配对** `pair_grand_staves()`：
- 相邻两个五线谱系统配对：偶数=高音谱，奇数=低音谱
- 验证：两谱系之间的间隙 < 高音谱高度 × 4（过远的不配对）

**输出**：`grand_staff_pairs` = `[(treble_system, bass_system), ...]`，每个 system = `[y1, y2, y3, y4, y5]`


### 第 3 阶段：小节线检测 (`symbol_detection.py`)

**方法**：双策略检测
1. **模板匹配**：用 `bar.jpg` 模板，在每个谱系的区域内多尺度匹配（理想缩放 ± 15%），阈值 0.50
2. **形态学后备**：垂直形态学开运算（核高度 = `谱系高度 × 0.8`），检测垂直连通域，宽度 < `dy × 0.8` 且高度 > `谱系高度 × 0.5`
3. 两种检测结果合并，去重（间距 < `dy × 3` 的取一个）
4. 去掉谱号区域和右边缘附近的误检
5. 上下两行谱的小节线合并对齐

**输出**：每个大谱表的小节线 x 坐标列表


### 第 4 阶段：音符头检测 (`template_matching.py`)

4 步检测策略：

**Step 0 — 排除区域**：
- 用谱号和升降号模板（treble_clef, bass_clef, sharp_1, flat_1 等）在每个谱系做多尺度匹配
- NMS（重叠阈值 0.3）→ 标记排除区域，避免将谱号边缘误识为音符

**Step 1 — 形态学检测（主要方法）**：
- 椭圆形态学开运算核（半径 = `dy × 0.65`），提取椭圆形黑色区域
- 连通域分析 → 按面积 (`dy² × 0.2 ~ dy² × 6.0`)、长宽比 (`0.5 ~ 2.2`)、凸度 (`> 0.45`) 过滤
- **和弦拆分**：若连通域高度 > `1.6 × dy`，进行垂直投影分析，在投影波谷处拆分为多个音符
  - 平滑投影曲线，检测局部极大值
  - 过滤弱峰值（< 最大峰值 × 65%）
  - 若拆分峰间距 < `dy × 0.3`，合并为同一音符

**Step 2 — 模板匹配（加线区域补充）**：
- 合成模板：椭圆音符头 (`dy × 1.3` × `dy × 0.85`)，带加线的音符头（1~3 条加线，上方/下方）
- 搜索区域：谱系上方 `5 × dy`、谱系下方 `5 × dy`（加线区域），以及谱系内部
- 多尺度匹配：缩放范围 `0.7 ~ 1.3`，阈值 0.55

**Step 3 — NMS + 去重**：
- 非极大值抑制（IoU 阈值 0.3）
- 中心距离去重（距离 < `dy × 0.4` 的保留得分最高者）

**输出**：`[(x, y, w, h, score), ...]` 音符头列表


### 第 5 阶段：音符分配到高/低音谱 (`note_assignment.py`)

**分配** `assign_notes_to_staves()`：
- 对每个音符，找最近的大谱表对（y方向距离 < `dy × 5`）
- 以高音谱第 5 线与低音谱第 1 线的中点为分界线
- 上方 → 高音谱，下方 → 低音谱
- 设置 `clef`, `pair_idx`, `system` 属性

**过滤误检** `filter_false_positive_notes()`：
- 根据音符的位置（距谱线距离）和检测得分进行层级过滤
- 谱线外越远，要求得分越高（例如高音谱下方 > `2dy` 的音符需得分 > 0.95）
- 不同谱号有不同的阈值策略


### 第 6 阶段：升降号检测 (`symbol_detection.py`)

**全局检测** `detect_accidentals_global()`：
- 加载所有 `sharp_*`, `flat_*`, `natural_*` / `nature_*` 模板
- 搜索范围：每个谱系 y 方向 ± `5 × dy`，跳过左端 18% 谱号区域
- 7 级缩放（`理想缩放 × 0.7 ~ 1.35`），理想缩放 = `2.0 × dy / 模板高度`
- 匹配阈值 0.60
- NMS 半径 = `dy × 0.8`

**关联到音符** `assign_accidentals_to_notes()`：
- 每个升降号关联到最近的、在其右侧的音符
- 约束：x 距离 `-0.5dy ~ 3.5dy`（升降号在音符左侧），y 距离 < `1.2 × dy`
- 按得分从高到低处理，每个音符只关联一个升降号

**输出**：`accidentals_map` = `{(note_cx, note_cy): '#' / 'b' / 'n'}`


### 第 7 阶段：休止符检测 (`symbol_detection.py`)

- 模板：`stop_4.jpg`（四分休止符 = 1 拍），`stop_8.jpg`（八分休止符 = 0.5 拍）
- 缩放基准：四分休止符 = 谱系高度 × 0.5，八分 = 谱系高度 × 0.35
- 匹配阈值：四分 0.55，八分 0.50
- NMS：x 方向 `2 × dy`，y 方向 `8 × dy`

**后处理过滤**（`main.py` `_filter_rests()`）：
- 去掉靠近小节线的误检（距离 < `3 × dy`）
- 去掉与音符重叠的误检（x < `1.5 × dy` 且 y < `2 × dy`）

**小节内清洗**（`note_unit.py` `_clean_rests_in_measure()`）：
- 去掉与和弦 x 坐标过近的休止符（距离 < `1.5 × dy`，避免符干被误识为休止符）
- 去重：x 距离 < `2 × dy` 的重复休止符只保留一个


### 第 8 阶段：符干追踪 (`stem_tracking.py`)

**输入**：音符头列表 + `music_symbols`（去谱线后的符号图）

**方法**：
1. 从每个音符头出发，在 `music_symbols` 图上追踪符干
2. **符干 x 定位**：在音符头边缘附近扫描列（搜索范围 = 音符头宽度/2 + `dy × 0.2`），取墨色密度最高的列作为 `stem_x`
3. **上下追踪**：从音符头中心向上/向下逐行扫描，允许间隙容忍（`dy × 0.3` 行），密度阈值 0.3
4. **方向判定**：较长一侧为符干方向，最短长度要求 = `dy × 1.5`

**输出**：每个音符附加 `stem_dir`（up/down/none）、`stem_x`、`stem_top`、`stem_bottom`、`stem_len` 属性


### 第 9 阶段：音符单元构建 + 小节分割 (`note_unit.py`)

**和弦分组**：
- 基于符干的分组：`stem_x` 距离 < `dy × 0.5` 的音符归为同一和弦（同一谱系，y 范围 < `4 × dy`）
- 后备分组：无符干时按音符头 x 距离 < `dy × 0.9` 分组

**时值估算**：
1. **空心符头检测**：检查音符头中心区域填充度，区分空心（二分/全音符）与实心
2. **比例间距法**：计算相邻和弦的 x 间距，按比例分配小节总拍数
3. **对齐到标准时值** `[0.25, 0.5, 1.0, 2.0, 4.0]`，若对齐后总和偏差 > 0.1，调整间距最大的事件补齐差额

**误检过滤**：
- 在 `music_symbols` 图上检查音符头中心区域的填充度
- 过滤被误检为音符头的升降号等符号（填充度不满足阈值）

**弱起小节检测**：
- 第一小节中所有事件集中在右侧 60% → 判定为弱起
- 所有音符时值设为八分音符，首部添加休止符

**小节分割**：
- 合并和弦事件和休止符事件 → 按 x 排序
- 用小节线 x 坐标切分为各小节
- 对每小节执行休止符清洗（去掉与和弦 x 坐标过近的休止符，去重）


### 第 10 阶段：简谱格式化 (`jianpu_formatter.py`)

**音高转换** `format_note()`：
- 调用 `y_to_jianpu()` 将 y 坐标转为简谱数字 + 八度后缀
- 升降号前缀处理，含小节内持续逻辑：
  - 升降号附着到音符后，对同一音高的后续音符持续生效
  - 还原号(n)取消持续效果
  - 持续距离限制 = `10 × dy`（超出范围视为自然消失）

**和弦格式** `format_chord()`：
- 单音 → `"3'"`
- 多音 → `"[1 3 5]"`

**时值后缀**：
- 1.0 拍（四分音符）→ 无后缀
- 0.5 拍（八分音符）→ `/2`
- 0.25 拍（十六分音符）→ `/4`

**休止符格式**：
- 1.0 拍 → `0`
- 0.5 拍 → `0/2`
- 0.25 拍 → `0/4`
- 2.0 拍 → `0 0`
- 4.0 拍 → `0 0 0 0`

**小节格式**：`|事件1 事件2 ...|`

## 简谱记号约定

| 记号 | 含义 | 示例 |
|------|------|------|
| `1`-`7` | 基本音符 do-si（默认四分音符） | `1` = do |
| `0` | 休止符 | `0` = 四分休止 |
| `'` | 高八度（后缀，可叠加） | `1'` = 高音do，`1''` = 倍高音do |
| `,` | 低八度（后缀，可叠加） | `1,` = 低音do，`1,,` = 倍低音do |
| `#` | 升号（前缀） | `#4` = 升fa |
| `b` | 降号（前缀） | `b3` = 降mi |
| `[...]` | 和弦（同时发声） | `[1 3 5]` = C大三和弦 |
| `/2` | 八分音符（半拍） | `1/2` |
| `/4` | 十六分音符（四分之一拍） | `1/4` |
| `\|...\|` | 小节 | `\|1 2 3 4\|` |

## 泛化性分析

### 算法中与图片无关的泛化设计

本项目的所有像素级参数均基于 `dy`（谱线间距）动态计算，而非硬编码像素值。`dy` 在第 2 阶段自动检测，使得算法能适应不同分辨率和大小的乐谱图片。

| 设计点 | 泛化能力 | 说明 |
|--------|----------|------|
| 谱线检测 | 强 | 基于水平投影 + 峰值聚类，不依赖特定分辨率 |
| 大谱表配对 | 强 | 基于相邻谱系间距比例判断 |
| 音符头检测 | 中 | 形态学核和模板尺寸均由 dy 推导，但需要黑白对比度良好 |
| 符干追踪 | 中 | 在去谱线图上追踪，但受符干倾斜和断裂影响 |
| 符干追踪 + 间距法 | 强 | 符干追踪改善和弦分组精度，间距法作为时值后备保证鲁棒性 |
| 升降号检测 | 中 | 多模板多尺度匹配有一定鲁棒性，但模板集有限 |
| 误检过滤 | 中 | 在music_symbols图上验证填充度，过滤被误检为音符的升降号等 |
| 休止符检测 | 弱-中 | 仅支持四分/八分休止符模板，且模板匹配容易产生误检 |

### 对不同乐谱的适用条件

**应当能正确处理的场景**：
- 分辨率 ≥ 150 DPI 的清晰扫描件或截图
- 标准钢琴大谱表（高音谱号 + 低音谱号）
- 2/4, 3/4, 4/4 等常见拍号
- 以四分、八分、十六分音符为主的节奏

**可能失败的场景**：
- 图片倾斜 > 2-3 度（谱线检测依赖水平投影）
- 低对比度、模糊、或有大量噪点的图片
- 手写乐谱（模板匹配假设印刷体符号）
- 非钢琴谱（单行谱、管弦总谱等非大谱表排版）
- 包含连线(tie/slur)、附点、三连音、装饰音的复杂节奏
- 调号中的升降号（当前被跳过而非解析为调号）
- 二分音符/全音符的时值（当前支持但未充分测试）

### 关键阈值与其影响

以下是对精度影响最大的阈值，调优时应优先关注：

| 阈值 | 位置 | 当前值 | 过高 | 过低 |
|------|------|--------|------|------|
| 音符头匹配阈值 | `template_matching.py` `threshold` | 0.55 | 漏检音符 | 误检（谱号/升降号碎片） |
| 升降号匹配阈值 | `symbol_detection.py` | 0.60 | 漏检升降号 | 误检（噪点/符干） |
| 符干最短长度 | `stem_tracking.py` | `dy × 1.5` | 漏检短符干 | 噪声被误认为符干 |
| 和弦 stem_x 容差 | `note_unit.py` | `dy × 0.5` | 跨和弦合并 | 和弦内音符被拆分 |
| 休止符-和弦距离 | `note_unit.py` | `dy × 1.5` | 真休止符被删 | 假休止符残留 |

### 音高映射说明

**高音谱号**：标准 G 谱号映射，19 个位置覆盖 F3-C6，超出范围使用扩展网格外推。

**低音谱号**：代码中有两种模式，由 `pitch_detection.py` 中的 `USE_LEGACY_BASS` 开关控制：
- `True`（当前默认）：低音谱位置按高音谱方式读取（用于匹配本测试用例的 ground truth 标注方式）
- `False`：标准 F 谱号映射（A1-E4），用于需要真实音高输出的场景

切换方式：在 `pitch_detection.py` 中修改 `USE_LEGACY_BASS = False`。

## 已知局限与未来方向

### 当前局限

1. **休止符检测**：仅支持四分和八分休止符，模板匹配容易在小节线和符干附近产生误检
2. **时值检测**：符杠计数受谱线残留影响，同一和弦内不同音符可能得到不同的符杠数；间距法对排版不均匀的乐谱效果下降
3. **调号解析**：当前直接跳过谱号区域（左端 18-20%），不解析调号中的升降号
4. **拍号识别**：`beats_per_measure` 硬编码为 2.0，不自动识别拍号
5. **高级节奏**：不支持附点、连线(tie/slur)、三连音、延长记号
6. **单行谱**：仅支持钢琴大谱表，不支持独奏谱、合唱谱等

### 改进方向

- 增加拍号自动识别（检测谱号右侧的数字）
- 增加调号解析（检测谱号区域的升降号模式）
- 改进休止符检测（结合位置语义：谱系中心、无音符区域等先验）
- 使用深度学习替代模板匹配（符号分类、端到端识别）
- 扩展 ground truth 覆盖全部行，进行更全面的评估
